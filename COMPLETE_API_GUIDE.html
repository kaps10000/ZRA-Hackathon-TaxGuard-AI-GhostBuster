📄 OCR AI Service - Port 8000

Base URL: http://localhost:8000

Technology: FastAPI + Python + Tesseract OCR

Purpose: Document processing, OCR, data extraction, verification & risk assessment

⚠️ IMPORTANT: Method Requirements

❌ GET requests will return "Method Not Allowed"

✅ All OCR endpoints require POST method with multipart/form-data

✅ Only Health Check and Docs use GET

Health & Info

Health Check

GET
http://localhost:8000/health

Returns: Service status and component health

curl http://localhost:8000/health

API Information

GET
http://localhost:8000/
curl http://localhost:8000/

OCR Processing

Process OCR

POST
http://localhost:8000/api/ocr/process

Description: Extract text from document using OCR

Parameters (Form Data):

file file REQUIRED
Image or PDF file (PNG, JPG, JPEG, PDF, TIFF)
preprocess boolean OPTIONAL
Apply image preprocessing (default: true)
language string OPTIONAL
OCR language code (default: eng)
curl -X POST http://localhost:8000/api/ocr/process \ -F "file=@test-invoice.png" \ -F "preprocess=true" \ -F "language=eng"

Data Extraction

Extract Data

POST
http://localhost:8000/api/extract

Description: OCR + AI field extraction

Parameters:

file file REQUIRED
document_type string OPTIONAL
Values: "invoice", "customs", "receipt"
curl -X POST http://localhost:8000/api/extract \ -F "file=@test-invoice.png" \ -F "document_type=invoice"

Verification & Risk Assessment

Verify Document (Full Workflow)

POST
http://localhost:8000/api/verify

Description: Complete verification: OCR → Extraction → Verification → Risk Score

Returns:

  • Verification status (approved/flagged/rejected)
  • Risk score (0-100)
  • Risk level (low/medium/high)
  • Detected anomalies
  • Recommendations
curl -X POST http://localhost:8000/api/verify \ -F "file=@test-invoice.png"

Batch Verify

POST
http://localhost:8000/api/verify/batch

Description: Process multiple documents in one request

curl -X POST http://localhost:8000/api/verify/batch \ -F "files=@test-invoice1.png" \ -F "files=@test-invoice2.png"
