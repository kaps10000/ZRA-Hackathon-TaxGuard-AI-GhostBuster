📄 OCR Processing

3. Process OCR

POST
http://localhost:8000/api/ocr/process

Description: Extract text from document using OCR

Parameters (Form Data):

file file REQUIRED
Upload image or PDF file (PNG, JPG, JPEG, PDF, TIFF)
preprocess boolean OPTIONAL
Apply image preprocessing (default: true)
language string OPTIONAL
OCR language code (default: eng)
curl -X POST http://localhost:8000/api/ocr/process \ -F "file=@/home/kaps100/ZRA-Hackathon-TaxGuard-AI-GhostBuster/ocr-backend/test-invoice.png" \ -F "preprocess=true" \ -F "language=eng"

Returns:

  • Extracted text
  • OCR confidence score
  • Word count
  • Processing time
  • Text regions

🔍 Data Extraction

4. Extract Data

POST
http://localhost:8000/api/extract

Description: Extract structured fields from document (OCR + AI extraction)

Parameters (Form Data):

file file REQUIRED
Upload invoice, customs, or receipt document
document_type string OPTIONAL
Document type hint: "invoice", "customs", "receipt" (auto-detect if not provided)
curl -X POST http://localhost:8000/api/extract \ -F "file=@/home/kaps100/ZRA-Hackathon-TaxGuard-AI-GhostBuster/ocr-backend/test-invoice.png" \ -F "document_type=invoice"

Returns:

  • Document type (classified)
  • Extracted fields (dates, amounts, IDs, etc.)
  • Confidence scores for each field
  • Validation errors
  • Raw OCR text

🔬 Document Verification

5. Verify Document (FULL WORKFLOW)

POST
http://localhost:8000/api/verify

Description: Complete verification: OCR → Extraction → Verification → Risk Assessment

Parameters (Form Data):

file file REQUIRED
Upload document for verification
document_type string OPTIONAL
Document type hint for better accuracy
curl -X POST http://localhost:8000/api/verify \ -F "file=@/home/kaps100/ZRA-Hackathon-TaxGuard-AI-GhostBuster/ocr-backend/test-invoice.png"

Workflow Steps:

  1. Step 1/3: OCR Processing
  2. Step 2/3: Data Extraction
  3. Step 3/3: Verification & Risk Assessment

Returns:

  • Verification Status: approved / flagged / rejected
  • Risk Score: 0-100 (0 = highest risk, 100 = lowest risk)
  • Risk Level: low / medium / high
  • Anomalies: List of detected issues
  • Recommendations: Suggested actions
  • Extracted Fields: All document data
  • OCR Confidence: Text extraction quality

📦 Batch Processing

6. Batch Verify

POST
http://localhost:8000/api/verify/batch

Description: Process multiple documents in a single request

Parameters (Form Data):

files file[] REQUIRED
Upload multiple files (repeat 'files' key for each file)
document_type string OPTIONAL
Document type hint for all files
curl -X POST http://localhost:8000/api/verify/batch \ -F "files=@/home/kaps100/ZRA-Hackathon-TaxGuard-AI-GhostBuster/ocr-backend/test-invoice.png" \ -F "files=@/home/kaps100/ZRA-Hackathon-TaxGuard-AI-GhostBuster/ocr-backend/test-invoice.jpg"
💡 Postman Tip

In Postman, add multiple form-data entries with key files and select different files for each.

Returns:

  • Total files processed
  • Array of results with:
    • Filename
    • Success status
    • Risk score & level
    • Extracted fields
    • Error messages (if failed)

🚀 Postman Setup Guide

Step 1: Import Collection

  1. Open Postman
  2. Click Import button
  3. Select file: ZRA_TaxGuard_API_Collection.postman_collection.json
  4. Click Import

Step 2: Configure File Upload

  1. Select an endpoint (e.g., "Verify Document")
  2. Go to Body tab
  3. Select form-data
  4. For the file key:
    • Change type dropdown to File
    • Click Select Files
    • Browse to test file location
  5. Click Send
⚡ Common Issues
  • "Method Not Allowed" → Change method from GET to POST
  • "Field required" → Make sure file is uploaded in form-data
  • "Invalid file type" → Use PNG, JPG, JPEG, PDF, or TIFF files only
  • Connection refused → Check if Docker containers are running
